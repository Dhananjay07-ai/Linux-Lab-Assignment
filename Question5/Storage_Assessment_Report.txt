---Command 4: df -i (Inode Utilization)---
Command: df -i
Output:
  /dev/disk3s1s1: 478724992 total inodes, 24552656 used,
                  124066088 free (17% iused)
  /dev/disk3s5:   478724992 total inodes, 301280216 used,
                  124066088 free (71% iused)
Observation: Inode usage mirrors disk usage percentages. The Data
volume has 71% of inodes used (301 million out of 478 million).
Inode exhaustion would prevent new file creation even if disk space
is available, so monitoring inode usage is as important as disk space.

---Storage Health Observations---
1. Total physical storage: 251 GB SSD (Apple internal)
2. Main data volume: 71% used - approaching high utilization
3. Home directory alone consumes 70GB
4. Inode utilization at 71% on Data volume - should be monitored
5. Recovery partition (1.3GB) and Preboot (9GB) are healthy
6. VM volume at 7% indicates low swap usage - good system health
7. Root filesystem at only 17% used - system files are well managed

---Recommendations for Improving Storage Utilization---
1. Clean Downloads folder - likely contains large unused files
2. Remove unused development tools and project files to free space
3. Use macOS Storage Management (Apple menu > About This Mac >
   Storage) to identify large files
4. Archive old projects to external storage or cloud
5. Clear cache files periodically to reclaim disk space
6. Monitor Data volume closely as it is at 71% - consider cleanup
   before it reaches 85%+ which can impact performance
7. Inode usage at 71% should be reviewed if many small files exist
